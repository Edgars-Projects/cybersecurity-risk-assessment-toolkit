# Incident Response Plan

This document outlines basic steps for responding to cybersecurity incidents.

## Step 1: Identification
Identify suspicious activity such as:

- unusual login attempts
- ransomware alerts
- unknown software installations

## Step 2: Containment
Immediately isolate affected systems to prevent spread.

Actions may include:

- disconnecting devices from network
- disabling compromised accounts

## Step 3: Investigation
Determine:

- how the breach occurred
- what systems were affected
- what data may be compromised

## Step 4: Recovery
Restore systems using clean backups and apply security patches.

## Step 5: Lessons Learned
Document the incident and improve security controls to prevent future attacks.
